# AE漏报风险核查 Skill

临床试验实验室检查/生命体征/心电图异常AE漏报风险自动核查工具。

## 目的

在临床试验数据清理和日常医学监察中，系统性地筛查实验室检查、生命体征、心电图等指标的基线-基线后异常变化，匹配CTCAE分级标准，评估是否存在不良事件（AE）漏报风险，生成交互式HTML核查报告。

## 适用场景

- 日常医学监查 — 受试者安全数据回顾
- 数据库锁库前数据清理 — 识别需补充医学判断的实验室异常
- CFDI现场核查准备 — 系统性筛查AE漏报风险

## 使用方法

### 触发词

在AI Agent（WorkBuddy、Claude Code、Codex、OpenClaw等）中，输入包含以下关键词的消息即可激活：

- `AE漏报`
- `实验室异常`
- `实验室检查异常`
- `AE漏报风险`

### 需要上传的文件

| 文件 | 是否必需 | 格式 | 说明 |
|-----|---------|------|------|
| Data Listing | ✅ 必需 | .xlsx | EDC系统导出的全量数据表（如FormExcelAllVersion格式） |
| CTCAE分级文件 | ✅ 必需 | .pdf | CTCAE v5.0 或 v6.0，中英文均可 |
| 受试者分组表 | 可选 | .xlsx | 包含受试者所在治疗组信息 |

### 工作流

激活Skill后，Agent会按以下流程引导您：

1. **文件上传** → 提示上传Data Listing、CTCAE分级文件、受试者分组表
2. **CTCAE版本确认** → 提醒确认是v5.0还是v6.0
3. **构建模式确认** → 首次全量构建 or 增量更新（需上传既往HTML报告，且data listing需为最新全量版）
4. **阈值确认** → 对于CTCAE不涉及的指标，异常判定阈值默认为较基线变化≥20%
5. **基线确认** → 基于方案确认基线的定义（筛选期/D1/其他）
6. **数据解析** → 自动识别Excel各Sheet的类型和结构，从「数据节」列优���提取访视信息
7. **CS解释验证** → 自动检测CS记录是否有临床意义解释，若缺失则提醒用户确认解释列位置
8. **CTCAE匹配** → 自动匹配检测指标对应的CTCAE条目
9. **风险分析** → 按规范流程筛选异常并分级
10. **报告生成** → 生成交互式HTML报告

## 报告内容

### 报告结构

生成的HTML报告包含以下元素：

- **项目头部**：报告标题、CTCAE版本及覆盖率、非CTCAE阈值、生成日期、数据来源
- **中心Tab切换**：按研究中心分Tab，各中心独立展示
- **汇总面板**：高风险/关注/低风险数量、异常受试者数、总项次
- **快速筛选**：按受试者编号/风险级别一键筛选
- **指标导航**：按检测类别分组（血常规/血生化、尿常规、生命体征、心电图、体重等），点击跳转
- **指标详情区**：按高→中→低风险排序，每人-指标组合展示为详细卡片：
  - 受试者编号 / 指标名称
  - 治疗分组标签（如有分组表）
  - 风险标签 + 异常计数 + CTCAE升级计数 + AE/MH解释状态
  - 相关AE/MH信息面板（含严重程度/与药物关系）
  - 基线信息（访视/结果/单位 + 基线状态）
  - 判读结论
  - 10列数据表格：访视（含V{n}子标签）/ 日期（MM-DD）/ 结果 / 单位 / 参考范围 / 方向 / CTCAE分级 / 临床意义 / 临床意义解释 / 较基线变化
  - 基线数据行：浅蓝色背景

### 风险分级标准

| 风险等级 | 定义 |
|---------|------|
| 🔴 **高风险** | 临床意义评价为"异常无临床意义"(NCS)或未评价，且无可解释的AE/MH记录 |
| 🟡 **关注** | AE/MH记录不足以充分解释异常程度、或NCS/CS评价不一致 |
| 🟢 **低风险** | 已有合理解释的AE/MH记录、或研究者评估为"正常"但技术性超出参考范围 |

### 检测指标覆盖

- **血常规**：白细胞、中性粒细胞、淋巴细胞、单核细胞、嗜酸性/碱性粒细胞、红细胞、血红蛋白、血小板等
- **血生化**：ALT、AST、ALP、TBil、DBil、ALB、TC、TG、GLU、Cr、BUN、电解质等
- **尿常规**：尿白细胞、尿红细胞、尿蛋白、尿糖、尿酮体等
- **生命体征**：收缩压、舒张压、体温、脉搏、呼吸频率
- **心电图**：QTcF、QT间期、心率、PR间期、QRS
- **体重**：基于CTCAE百分比变化分级（仅在基线后有多次测量时纳入）

## 核心风险判定逻辑

```
步骤0：排除规则
  ├─ 全研究仅一次检测的指标 → 不纳入
  └─ 基线异常但基线后恢复正常 → 不列为风险项

步骤1：异常筛选
  ├─ 基线正常 + 基线后超出参考范围 → 新发异常
  └─ 基线异常 + 基线后CTCAE升级/非CTCAE变化≥阈值 → 异常加重

步骤2：AE/MH关联判断
  ├─ 解析"临床意义解释"/"动态链接"中引用的AE/MH编号
  ├─ 支持加号分隔（MH+4+术语）和空格分隔（MH4）两种EDC格式
  ├─ 智能匹配AE/MH术语与异常指标的相关性
  ├─ 排除尿液-血液非同一样本系统的错误关联
  └─ 体重异常区分增加/降低方向匹配

步骤3：风险分级
  ├─ NCS/未评价 + 无解释 → 高风险
  ├─ 有解释但关联不足 → 关注
  └─ 合理解释/研究者评估正常 → 低风险
```

## 安装方法

### 安装到WorkBuddy

1. 下载 `ae-risk-assessment.zip`
2. 在WorkBuddy中使用 `/install-skill` 命令导入
3. 或手动解压到 `~/.workbuddy/skills/ae-risk-assessment/`

### 安装到其他Agent

将ZIP包解压到Agent的技能目录中。确保Agent安装了必要的Python依赖：

```bash
pip install openpyxl
```

可选：安装 `pdfplumber` 以支持CTCAE PDF自动解析（否则使用内置v5.0映射表）。

## 关键特性

- **自适应解析**：不硬编码sheet名/列名/指标名，基于上传文件动态适配不同EDC导出格式
- **数据节优先**：自动检测「数据节」列获取访视信息，兼容多种EDC系统
- **CS验证**：自动检测异常有临床意义记录的临床意义解释完整性，提示缺失数据
- **智能排除**：筛选期/基线期后无后续数据的指标自动排除，基线异常→正常不列风险
- **V{n}提取**：从访视名称中提取访视编号，仅当有意义时显示
- **动态链接**：支持解析EDC「动态链接」列中的AE/MH引用
- **分组展示**：根据分组表在报告中显示治疗组标签

## 局限性说明

- CTCAE匹配基于内置映射表（覆盖v5.0主要指标），PDF自动解析为增强功能
- AE/MH关联判断基于关键词模糊匹配，非语义理解。建议用户审查关联逻辑
- 不替代医学判断，报告结果为辅助筛查建议
- 非CTCAE指标使用百分比变化阈值，可能产生假阳性
